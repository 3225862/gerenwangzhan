# 内容准备清单

## 上线前必须

- 网站名称。
- 默认语言。
- 姓名或个人品牌名称。
- 一句话定位。
- 简短个人介绍。
- 头像或可替代的品牌图形。
- 至少 1-3 个代表项目。
- 每个代表项目的名称、简介、时间、角色、图片或截图。
- 教育经历公开版本。
- 至少一个联系方式，推荐邮箱。
- 社交链接，至少 GitHub 或其他主要账号。
- 首页展示内容选择。
- 隐私公开范围：姓名、头像、所在地、学校、简历、邮箱是否公开。
- 简历 PDF，或确认第一版不提供下载。
- 域名候选或已购买域名。
- 语言版本优先级：中文、日语、英语的上线顺序。

## 上线前建议

- 完整个人简介。
- 个人照片或工作照。
- 技能和研究方向列表。
- 语言能力说明。
- 个人经历亮点。
- 个人理念和长期目标。
- 更多项目资料。
- 项目 GitHub 链接、演示链接、相关文件。
- 3-5 篇初始文章或研究笔记。
- 作品与成果文件。
- 时间轴事件。
- 网站 Open Graph 分享图。
- SEO 默认标题和描述。
- 404 页面文案。
- 简历多语言版本。

## 上线后补充

- 更多文章。
- 更多项目复盘。
- 研究成果和报告。
- 演示文稿。
- 获奖、证书、公开活动。
- 多语言完整翻译。
- 更细的分类和标签。
- 更丰富的作品图片。
- 访问统计方案。
- 联系表单需求。
- 中国大陆部署或备案资料。

## P2 发布前

- [ ] 替换首页、个人资料和站点设置中的占位内容。
- [ ] 逐项确认姓名、所在地、邮箱、简历和社交链接的公开开关。
- [ ] 确认 Hero、CTA、区块顺序与显示状态。
- [ ] 确认推荐项目同时满足 `featured: true` 与 `draft: false`。
- [ ] 不使用未经本人确认的个人事实或图片。

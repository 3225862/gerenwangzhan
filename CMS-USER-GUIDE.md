# CMS 用户指南

## 进入后台

本地运行 `npm run dev` 后打开 `/admin`。先看“内容总览”，再从快捷入口进入 Keystatic 编辑。生产环境只有在网站所有者配置 GitHub mode 后才启用后台。

## 修改首页和关于我

从快捷入口选择“编辑首页”或“编辑关于我”。内容按 Hero、介绍、推荐项目、当前状态、联系方式，以及基本信息、头像、简介、教育、技能、兴趣、联系方式和长期目标整理。填写后先保存草稿，再打开前台检查。

## 新增项目或文章

选择“新建项目”或“新建文章”。先填写标题和摘要，确认 slug、封面、标签、日期和 SEO 字段，再决定保留草稿或发布。草稿不会进入公开页面。项目的相关项目使用已有 slug；不存在的内容不要手动猜写。

## 图片、PDF 和视频

图片只使用 jpg、jpeg、png、webp，单张不超过 2MB。PDF、PPT、Word 和视频目前填写经过确认的外部链接，避免大型文件进入 Git。视频链接支持后续接入 YouTube、Bilibili、腾讯视频或自定义播放器适配，但当前前台不会伪造播放器。

## 保存、发布、恢复和删除

建议先保存草稿并检查前台，确认无误后取消草稿发布。恢复内容使用 Git 历史或仓库备份，不需要修改代码。删除前应先确认是否仍有页面引用；当前 Keystatic 删除操作依赖其编辑界面，软删除和二次确认仍是后续增强项。

## 仍可能需要技术协助的事项

GitHub OAuth、域名、部署账号、批量媒体库、自动压缩/裁剪、PDF 对象存储和复杂播放器尚未在本阶段伪装成已完成能力。

图片直接在首页、关于我、项目或文章的图片字段中选择。不要上传大型原图或视频。视频使用 Bilibili、YouTube 或其他确认过的外部链接；小型 PDF、PPT、Word 可以放在 `public/downloads`，大文件放到外部存储后填写链接。

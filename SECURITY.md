# 安全设计

## 1. 威胁模型

P0 安全边界：Keystatic 仅启用 local mode，后台只用于本机开发；P0 不连接 GitHub，不使用生产密钥，不提供用户注册或公开写入接口。正式部署前必须切换 GitHub mode 并完成身份、环境变量和后台访问控制配置。

主要资产：

- Git 仓库。
- 管理后台账号。
- 内容文件。
- 图片和附件。
- 环境变量。
- 域名和 DNS。
- 自动部署权限。

主要威胁：

- 未授权访问后台。
- 管理员凭证泄露。
- 恶意文件上传。
- XSS。
- CSRF。
- 暴力登录。
- 后台接口滥用。
- 环境变量泄露。
- Git 仓库泄露。
- 第三方依赖漏洞。
- 内容误删除。
- 备份丢失或无法恢复。

## 2. 身份验证

- 后台必须登录后访问。
- P0 本地模式只允许本机开发使用；P9 才启用 GitHub 身份登录。
- 管理员账号必须开启双因素认证。
- 不共享管理员账号。
- 离开设备时退出登录或锁屏。

## 3. 管理后台保护

- `/admin` 不进入 sitemap。
- robots.txt 禁止抓取 `/admin/`。
- 后台页面添加 `noindex,nofollow`。
- 生产环境可增加访问控制，如平台保护、Cloudflare Access、基本认证或 IP 限制。
- 后台入口不在公开导航展示。

## 4. 文件上传安全

- P0 图片白名单为 jpg、jpeg、png、webp；单张上限目标 2 MB，推荐最长边 2400 px；SVG 默认不允许。
- 禁止可执行文件、HTML、脚本、压缩包、大型 PDF、Office 文件和视频进入 Git。
- Keystatic 只写入配置指定目录；前端只展示生成的公开路径，不接受用户输入的任意文件路径，避免路径穿越。
- P0 不承诺清理原始 EXIF；P8 增加上传后清理/去定位信息策略。
- P6/P9 以 `MediaAsset` 的路径、alt、尺寸、类型和来源字段作为对象存储迁移边界。
- 构建期检查会验证扩展名仅为 jpg、jpeg、png、webp，单文件不超过 2 MB，引用位于 `/uploads/projects/`，不存在 `../` 路径穿越，且引用文件必须存在。
- 当前尚未自动清理 EXIF；图片像素尺寸检查和更完整的媒体处理列入 P6/P8，不应将当前检查理解为已完成 EXIF 或尺寸处理。
- 图片需生成安全预览和 alt 文本。

## 5. XSS

- Markdown/MDX 渲染时限制危险 HTML。
- 外链添加 `rel="noopener noreferrer"`。
- 用户可编辑内容不直接插入原始 HTML。
- 如需嵌入 iframe，使用白名单并提供中国大陆不可访问时的替代链接。

## 6. CSRF

- 如果后台通过 GitHub/Keystatic 写入，依赖其身份和写入流程。
- 若后续自建 API，必须添加 CSRF token、SameSite Cookie 和来源校验。

## 7. 密钥管理

- 密钥只放在部署平台环境变量或密码管理器。
- `.env` 不提交 Git。
- 提交 `.env.example` 只放变量名和说明。
- 定期轮换部署 token。
- 不在前端代码、公开文档或截图中暴露密钥。

## 8. 依赖安全

- 使用锁文件。
- 直接依赖使用明确版本号，不使用 `latest`、`*`、`^` 或 `~`。
- 升级采用“修改明确版本 -> 更新 lockfile -> `npm ci` -> 全套检查”的受控流程。
- 定期检查依赖漏洞。
- 使用自动化依赖更新工具。
- 对大版本升级先在本地或预览环境验证。
- 避免引入不必要的第三方脚本。

## 9. 备份安全

- Git 仓库开启私有或明确控制公开范围。
- 备份文件加密或放在可信云盘/硬盘。
- 环境变量单独备份，不和公开内容包混放。
- 定期验证备份可恢复。

## 10. 日志

- 初期不收集访客敏感信息。
- 如启用访问统计，优先选择隐私友好、自托管或中国大陆可访问的方案。
- 不记录后台密码、token、完整环境变量。
- 错误日志需避免包含隐私内容。

## 11. 恢复方案

- 内容误删：从 Git 历史恢复单个文件。
- 网站构建失败：回滚到上一个成功 commit。
- 图片丢失：从媒体备份恢复。
- 密钥泄露：立即撤销旧密钥，生成新密钥，检查访问日志。
- 域名/DNS 异常：登录域名注册商和 DNS 服务商恢复记录。

## 12. 上线前安全检查清单

- [ ] 后台需要登录。
- [ ] 管理员开启双因素认证。
- [ ] `/admin` 不进 sitemap。
- [ ] robots.txt 禁止 `/admin/`。
- [ ] 页面无前端密钥。
- [ ] `.env` 未提交 Git。
- [ ] 上传类型和大小受限。
- [ ] 依赖无高危漏洞。
- [ ] 第三方脚本已审查。
- [ ] 备份流程已演练。
- [ ] 404、错误页不暴露堆栈信息。
- [ ] CDN/托管平台启用 HTTPS。

## P2 公开内容边界

个人资料中的法定姓名、所在地、邮箱和简历均由独立公开开关保护，默认关闭。页面不输出未公开字段，也不接受任意 HTML、脚本或外部资源作为内容。当前图片仍由构建期路径和扩展名规则约束；P2 尚未自动清理 EXIF，超大图片尺寸处理继续列入 P6/P8，不应视为已完成。

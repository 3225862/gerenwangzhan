# 后期维护说明

这份文档面向非技术用户，说明网站以后怎么更新和维护。

## 1. 我以后如何登录后台

P0 本地测试后台地址是：

```text
http://127.0.0.1:4321/admin
```

P0 使用本地模式，不需要登录账号；开发时 `/admin` 会转到 `/keystatic`。当前生产构建的 `/admin` 显示“后台尚未启用”，不会跳转到不存在的后台。正式上线在 P9 切换 GitHub mode，届时使用本人 GitHub 账号登录，并要求 GitHub 开启双重验证。

## 2. 如何新增项目

1. 在项目目录运行 `npm install`，再运行 `npm run dev`。
2. 进入“项目管理”。
3. 点击“新建项目”。
4. 填写项目名称、副标题、简介、正文、时间、角色、技术或方法。
5. 上传封面图和项目图片。
6. 添加标签。
7. 选择是否推荐到首页。
8. 先保存为草稿。
9. 保存后将 `draft` 改为关闭，构建时才会生成公开详情页。

P0 只有测试项目管理，不包含正式文章、首页区块和完整媒体库。

## 3. 如何发布文章

1. 进入“文章管理”。
2. 点击“新建文章”。
3. 填写标题、摘要、正文、分类和标签。
4. 上传封面图。
5. 填写 SEO 标题和描述。
6. 保存草稿。
7. 检查排版、链接和图片。
8. 发布文章。

文章编辑器应支持标题、段落、图片、引用、列表、链接、表格、代码块、分隔线和小标题。

## 4. 如何上传图片

1. 在编辑测试项目时选择上传图片。
2. 只使用 jpg、jpeg、png 或 webp；SVG、GIF 默认不允许。
3. 单张最大 2 MB，引用必须位于允许的 `/uploads/projects/` 目录。
4. 每张图片都填写替代文字，方便搜索引擎和无障碍阅读。
5. 不上传可执行文件、HTML、脚本、压缩包、大型 PDF、Office 文件、视频或敏感图片。

当前检查不会自动清理 EXIF，也不会自动判断图片像素尺寸；这两项列入 P6/P8。

## 5. 如何修改首页

1. 进入“首页内容管理”。
2. 修改首页标题、简介、代表项目、最新文章、代表经历、技能方向。
3. 调整区块顺序。
4. 保存并预览。
5. 确认后发布。

## 6. 如何回滚错误内容

如果只是草稿写错，直接在后台修改即可。

如果已经发布：

1. 回到后台修改内容并重新发布。
2. 如果误删重要内容，从 Git 历史恢复。
3. 如果整个网站发布出错，回滚到上一个正常版本。

正式开发阶段会把“恢复步骤”做成更清晰的操作手册。

## 6. 如何使用公共视觉设置

P1 已提供明暗模式和移动端菜单。点击右上角主题按钮即可切换，选择会保存在当前浏览器；首次访问会跟随系统明暗偏好。窄屏幕会自动使用菜单按钮，按 Escape 可关闭菜单。

`/style-guide` 仅供内部验收使用，不是正式内容页面，也不应作为对外入口。

## 7. 如何备份

推荐备份三类内容：

- 网站代码和文字内容：保存在 Git 仓库。
- 图片和附件：保存在媒体目录或对象存储，并定期导出。
- 登录密钥和环境变量：保存在密码管理器，不放进 Git。

建议每月做一次完整导出，重要修改前额外备份。

## 8. 如何恢复

P0 内容直接保存在仓库内的 `src/content/projects` 和 `src/data`，图片保存在 `public/uploads/projects`。网站损坏时按顺序处理：

1. 确认域名和托管平台是否正常。
2. 回滚到上一个成功发布版本。
3. 从 Git 恢复内容文件。
4. 从图片备份恢复媒体文件。
5. 重新部署网站。

## 9. 如何升级依赖

依赖升级不需要频繁手动做。推荐：

- 每月检查一次普通更新。
- 发现高危安全漏洞时尽快更新。
- 大版本升级先在预览环境测试。
- 升级后检查首页、项目、文章、后台和构建是否正常。

## 10. 多久检查一次

- 每周：查看网站是否能正常打开。
- 每月：检查备份、依赖、安全提醒。
- 每季度：检查内容是否过期、简历是否需要更新。
- 每年：检查域名续费、部署成本、备案或平台策略变化。

## 11. 一年预计维护费用

- 最低成本方案：约 USD 10-30/年，主要是域名费用。
- 推荐方案：约 USD 20-150/年，取决于图片存储和托管选择。
- 中国大陆正式部署：约 RMB 300-2000+/年，取决于云服务、CDN 和流量。

## 12. 可以自动完成的维护

- 内容保存后自动部署。
- 图片构建期压缩。
- Sitemap 和 RSS 自动生成。
- 依赖安全提醒。
- 定期备份提醒。

## 13. 需要人工完成的维护

- 判断哪些内容公开。
- 写文章、更新项目和简历。
- 审核隐私信息。
- 处理域名续费。
- 确认重要升级。
- 备案和云平台实名认证。

## 14. 日常注意事项

- 不要把密码、密钥、个人证件上传到公开内容。
- 发布前检查手机端显示。
- 上传图片时填写说明文字。
- 对外公开的联系方式优先使用邮箱和社交账号。
- 不默认公开电话号码、家庭地址等高风险隐私信息。

## 15. 依赖升级

依赖版本已固定。升级时先备份或创建分支，只修改明确版本号，再运行 `npm install --package-lock-only`、`npm ci`、lint、类型检查、测试、内容检查、外部依赖检查和 build。所有检查通过后再提交，不使用 `latest`、`*`、`^` 或 `~`。

## P2 内容维护

本地运行 `npm run dev` 后进入 `/admin`，编辑“首页内容”“个人资料”“网站设置”并保存；保存结果直接写入 Git 可备份的 YAML/内容文件。发布前检查占位标记和隐私开关，避免将默认邮箱或个人资料误公开。P2 只提供预设区块的顺序与可见性调整，不支持自由拖拽页面设计。

图片使用 Keystatic 的图片字段，建议使用 webp、jpg 或 png 且单张小于 2MB；不要把大型视频或附件提交到 Git。视频使用外部链接，小型 PDF/PPT/Word 可放在 `public/downloads`，大文件使用外部存储。内容维护优先通过 `/admin` 和 `/keystatic` 完成，不要求编辑 YAML。

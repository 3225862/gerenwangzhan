# 部署方案比较

## 总体建议

初期推荐方案 A 或 B：Astro 静态站 + Keystatic + Git 仓库 + 自动部署。这样不需要数据库，维护成本低，迁移路径清晰。

中国大陆正式部署需要 ICP 备案、国内 CDN、国内对象存储和更严格的内容合规准备，建议作为未来阶段处理。

## 方案 A：最低成本

| 项目             | 方案                                             |
| ---------------- | ------------------------------------------------ |
| 服务器或平台     | Cloudflare Pages / GitHub Pages / Netlify 免费层 |
| 域名             | 自有域名，约 USD 10-20/年                        |
| DNS              | Cloudflare DNS 免费                              |
| CDN              | 托管平台自带 CDN                                 |
| 图片存储         | 初期 Git 仓库媒体目录                            |
| HTTPS            | 平台免费证书                                     |
| 自动部署         | Git push 自动构建                                |
| 备份             | Git 仓库 + 每月媒体导出                          |
| 预计月成本       | USD 0-2，不含域名                                |
| 预计年成本       | USD 10-30                                        |
| 优点             | 成本最低、简单、适合早期、无需数据库             |
| 缺点             | 中国大陆访问不稳定，后台依赖 Git/GitHub 访问     |
| 中国大陆访问风险 | 中到高，取决于平台线路和 GitHub 可用性           |
| 迁移难度         | 低                                               |

适合：早期上线、内容量小、预算优先。

## 方案 B：中日访问平衡

| 项目             | 方案                                                              |
| ---------------- | ----------------------------------------------------------------- |
| 服务器或平台     | Cloudflare Pages，或香港/日本/新加坡静态主机                      |
| 域名             | 自有域名                                                          |
| DNS              | Cloudflare DNS 或域名商 DNS                                       |
| CDN              | Cloudflare CDN，必要时评估亚洲节点表现                            |
| 图片存储         | Cloudflare R2、Backblaze B2、S3 兼容对象存储，或继续 Git 媒体目录 |
| HTTPS            | 免费证书                                                          |
| 自动部署         | Git 自动部署                                                      |
| 备份             | Git + 对象存储定期导出                                            |
| 预计月成本       | USD 0-10                                                          |
| 预计年成本       | USD 20-150                                                        |
| 优点             | 日本访问快，海外部署无需 ICP，成本可控                            |
| 缺点             | 中国大陆访问仍不能保证，部分服务可能偶发不可达                    |
| 中国大陆访问风险 | 中等                                                              |
| 迁移难度         | 低到中                                                            |

推荐作为第一版正式公开网站方案。若主要访问者在日本和国际网络，此方案性价比最好。

## 方案 C：中国大陆正式部署

| 项目             | 方案                                                       |
| ---------------- | ---------------------------------------------------------- |
| 服务器或平台     | 阿里云/腾讯云/华为云静态托管、轻量服务器或对象存储静态网站 |
| 域名             | 自有域名，完成 ICP 备案                                    |
| DNS              | 国内 DNS 或云厂商 DNS                                      |
| CDN              | 阿里云 CDN / 腾讯云 CDN / 华为云 CDN                       |
| 图片存储         | 阿里云 OSS / 腾讯云 COS / 华为云 OBS                       |
| HTTPS            | 云厂商证书或免费证书                                       |
| 自动部署         | GitHub Actions/GitLab CI/云效等推送到国内云                |
| 备份             | Git + OSS/COS 版本控制 + 定期导出                          |
| 预计月成本       | RMB 20-150+                                                |
| 预计年成本       | RMB 300-2000+，不含可能的人工备案时间                      |
| 优点             | 中国大陆访问体验最好，可接国内 CDN                         |
| 缺点             | 需要 ICP 备案，云平台配置复杂，合规要求更高                |
| 中国大陆访问风险 | 低，但取决于备案、CDN、内容合规和服务商                    |
| 迁移难度         | 中                                                         |

适合：明确需要中国大陆稳定访问、愿意完成备案、接受国内云维护成本。

## CMS 与部署关系

- Keystatic 低成本模式可保存到本地/Git，适合 A/B。
- 若中国大陆后台访问 GitHub 不稳定，方案 C 可能需要：
  - 使用国内 Git 托管镜像。
  - 后台部署在国内服务器。
  - 或改为 Headless CMS/自托管 CMS，但保持内容导出格式。

## 是否需要数据库

初期不需要数据库。推荐使用 Git 文件内容作为主要数据源。

未来只有在出现以下需求时才考虑数据库：

- 多管理员复杂权限。
- 访客注册。
- 评论、投稿、社区。
- 实时搜索或动态数据。
- 大规模媒体管理。

## 是否需要 ICP 备案

- 使用境外服务器或 Cloudflare Pages：通常不需要 ICP 备案，但中国大陆访问稳定性不可保证。
- 使用中国大陆服务器、国内 CDN 或国内对象存储绑定自有域名：需要 ICP 备案。

## 推荐结论

- 第一阶段公开上线：方案 B。
- 极低成本试运行：方案 A。
- 面向中国大陆稳定访问的正式版本：方案 C。

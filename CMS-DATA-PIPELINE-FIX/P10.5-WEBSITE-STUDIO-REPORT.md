# P10.5 Website Studio 报告

## 变得更容易的地方

- 后台品牌统一为 Website Studio，不再把用户引导到“CMS 配置”概念。
- `/admin` 现在是工作台入口，提供网站概况、待处理草稿、快速开始、媒体库和前台预览。
- `/admin/preview` 提供一键打开首页、关于我、项目、文章、时间轴和作品。
- `/admin/media` 集中说明媒体规则、外部大文件策略和当前能力边界。
- 项目和文章新增“草稿 / 已发布 / 已归档”状态字段；归档内容不进入公开列表。
- Keystatic 品牌、状态字段和关键中文说明已更新。
- 新增 `CMS-USER-GUIDE.md`，以网站所有者视角说明日常维护。

## 媒体库能力

当前媒体字段仍由 Keystatic 统一保存，图片可从项目、文章、首页和个人资料的对应字段进入。媒体检查页统一展示格式、2MB、危险文件、EXIF 和大文件策略。

## 尚未完成的真实能力

本阶段没有伪造服务端能力。拖拽/批量上传、媒体缩略图索引、搜索排序、复制链接、引用位置反查、删除二次确认、图片裁剪/压缩/WebP、PDF/Office 上传、视频播放器和对象存储仍需媒体服务或后端接口支持。文章正文目前仍是多行文本编辑器，不是完整 Notion 富文本编辑器。

## 首页、项目和文章

首页、项目、文章都有中文后台入口，预览入口不需要理解路由。Homepage 的区块显示和顺序仍由已有模型控制；本阶段没有改变前台视觉设计，也没有增加前台页面功能。

## 仍需代码的地方

真实 GitHub mode、服务端媒体库、富文本文档存储、自动保存/离开提醒、删除保护、软删除、视频/PDF 前台渲染和完整后台实时预览仍需要代码与部署环境支持。普通内容编辑、草稿管理和预览不需要打开 VSCode。

## 验证

已执行 `npm ci`、`npm run lint`、`npm run check`、`npm test`、`npm run check:external` 和 `npm run build`。类型检查无错误，仅有既有 Zod 4 API 弃用提示。

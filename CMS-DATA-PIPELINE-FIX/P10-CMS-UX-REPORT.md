# P10 CMS UX 升级报告

## 已完成

- 新增本地 CMS Dashboard `/admin`：统计已发布项目、草稿、已发布文章和已引用媒体。
- 增加快捷入口：编辑首页、关于我、项目、文章、网站设置和媒体检查。
- 增加发布提醒：草稿、图片规则、SEO/Alt/外链构建检查和当前 CMS mode。
- 新增 `/admin/media` 媒体规则页，统一说明图片限制、EXIF 状态和大文件策略。
- 新增面向非技术用户的 `CMS-USER-GUIDE.md`。
- 保持前台路由、内容格式、设计系统和公开数据边界不变。

## 比以前更简单的操作

用户可先从 `/admin` 总览进入目标内容，不必记忆 Keystatic 内部入口；首页、关于我、项目、文章和网站设置均有明确中文入口；媒体限制和草稿规则在后台入口可见。

## 当前真实能力边界

Keystatic 继续负责内容保存和集合编辑。拖拽上传、批量上传、裁剪、自动压缩、媒体引用反查、PDF/Office 上传、视频播放器、软删除和保存前可视化预览尚未接入服务端能力，本阶段通过后台说明页明确标出，没有伪装成完成。

## 仍需写代码的地方

真实 GitHub mode、服务端媒体库、PDF/Office 对象存储、视频播放器适配、完整删除确认/软删除、自动保存和真正的编辑预览仍需后续技术实现。普通内容维护不需要修改代码或 YAML。

## 验收

本阶段执行 `npm ci`、lint、类型检查、测试、外部依赖检查和 build；后台升级不改变公开页面生成逻辑。

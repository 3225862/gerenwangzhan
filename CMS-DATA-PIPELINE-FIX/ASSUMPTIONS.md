# 假设、已确认决定与决策记录

## 1. 已确认决定

- GitHub 私人仓库由网站所有者本人控制，初期仅一个管理员。
- 第一版接受 GitHub 身份登录；管理员 GitHub 账号必须启用双重验证。
- 默认语言为中文，预留 `/zh/`、`/ja/`、`/en/`；第一版中文完整，日语和英语先保留技术结构。
- 第一版使用境外低成本部署，不办理 ICP；中国大陆目标为尽量兼容。
- 初期只使用邮箱链接和社交链接，不做联系表单、注册、评论、投稿、支付和社区。
- 少量图片可入 Git；大型视频、PDF、Office 文件及压缩包不得入 Git；媒体模型必须可迁移对象存储。
- 年度基础预算目标为人民币 500 元以内，域名、仓库、内容和部署账号由本人控制。
- 后台提供可视化内容管理和预设区块排序，不追求自由拖拽页面设计。
- 关键公开页面不得依赖 Google Fonts、Google Analytics、Google Maps、Google reCAPTCHA、YouTube 等资源。

## 2. 决策记录

### D-001：CMS 前移到 P0

CMS 从原 P6 前移到 P0，以尽早验证内容模型、保存格式、图片路径和草稿规则，减少正式页面完成后的返工。

### D-002：本地 Keystatic 先行，GitHub mode 后置

P0 采用 local storage，便于本机验证；P9 再配置 GitHub mode 和登录。P0 不把 GitHub 密钥放入前端或仓库。

### D-003：语言不回退

未翻译内容不生成对应语言详情页，不静默回退到其他语言，避免访问者误以为页面已经完成翻译。

### D-004：媒体分阶段迁移

P0 允许少量受限图片进入 Git；大型附件禁止进入 Git。P6/P9 再根据规模迁移对象存储，并保留可导出的媒体元数据。

### D-005：生产后台默认关闭

本地开发使用 `KEYSTATIC_MODE=local`，`/admin` 跳转到 `/keystatic`。未启用正式 GitHub mode 的生产构建使用 `KEYSTATIC_MODE=disabled`，`/admin` 只显示后台尚未启用，不暴露内部配置或无效跳转。

### D-006：内容和媒体构建前检查

Project schema、slug 唯一性、文件名映射、图片扩展名、2 MB 上限、允许目录、路径穿越和文件存在性由共享校验模块及构建前脚本检查。EXIF 清理和像素尺寸处理尚未实现，列入 P6/P8。

## 3. 当前不阻塞 P0 的待确认事项

- 最终域名、注册商和 DNS 托管位置。
- 生产静态托管平台及是否需要香港、日本或新加坡节点。
- 公开姓名、头像、所在地、学校、经历、简历和邮箱的具体隐私范围。
- 文章、作品和项目中涉及的版权、保密或不公开内容。
- GitHub mode 的 OAuth App 回调地址、仓库名和部署平台环境变量。

这些问题在对应路线阶段确认，不再阻塞 P0。

## P2 假设与边界

- 当前个人资料、首页文案和图片均视为占位内容，未经本人确认不作为真实事实发布。
- 长文本先以可版本控制的多行 Markdown-compatible 文本保存，P2 不实现复杂文档渲染。
- 推荐项目复用 P0 collection，完整项目系统留在 P3。
- 隐私公开开关默认为关闭；公开联系方式由网站所有者逐项确认。

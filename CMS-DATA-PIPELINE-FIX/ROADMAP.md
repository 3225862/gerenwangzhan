# 开发路线图

本路线图以“先验证内容进入后台、保存、读取、构建和发布规则，再逐步扩展页面”为原则。当前只执行 P1，完成后停止。

## P0：项目基础、内容模型和最小 CMS 垂直切片

- 目标：验证 `/admin` -> Keystatic -> Git 兼容文件 -> Astro -> 公开测试页的完整链路。
- 任务：初始化 Git/Astro、严格 TypeScript、锁定 npm、格式化/lint、环境变量模板；配置 Project collection 和 SiteSettings singleton；实现本地 CMS、图片限制、草稿过滤、最小列表/详情页和自动化检查。
- 交付物：可启动项目、最小后台、测试项目内容、测试图片路径规则、测试和 README。
- 验收：本地可启动；`/admin` 可进入；可创建/修改测试项目；内容写入仓库；已发布项目可见；草稿不生成公开页；构建、类型、lint、测试和外部依赖检查通过。
- 非目标：完整首页、完整项目系统、完整文章系统、正式视觉设计、多语言翻译、生产部署。

## P1：设计系统、公共布局和导航

- 目标：建立正式视觉基础和公共页面骨架。
- 任务：设计 token、字体和图标本地化、响应式布局、导航、页脚、语言结构占位。
- 前置条件：P0 通过。
- 当前交付：CSS token、SiteLayout、Header、桌面/移动导航、Footer、主题/语言切换、基础组件和内部 `/style-guide`。
- 非目标：完整首页、关于我、正式项目/文章系统、时间轴、作品系统、联系表单和后台美化。

## P2：首页、个人资料和关于我

- 目标：实现中文正式首页、个人资料和关于我。
- 前置条件：P1 通过，公开内容清单确认。

## P3：项目系统

- 目标：在 P0 测试链路之上扩展正式项目列表、筛选、详情、标签和推荐。

## P4：文章系统

- 目标：实现文章列表、详情、分类、标签、目录、阅读时间和 RSS。

## P5：经历、教育、时间轴、作品与成果

- 目标：实现个人经历、教育、时间轴、文档/研究/设计/视频/报告/演示稿等展示。

## P6：完善 CMS 后台体验与媒体管理

- 目标：完善可视化字段、预设区块顺序、媒体清单、批量维护、对象存储迁移边界和 GitHub mode。
- 原因：后台最小链路前移到 P0，以尽早发现内容模型和保存方式问题。

## P7：多语言

- 目标：完成 `/zh/`、`/ja/`、`/en/` 技术结构；中文完整，日语/英语按内容计划逐步补齐。
- 规则：未翻译内容不生成对应语言详情页，不静默回退其他语言。

## P8：SEO、性能、安全与中国大陆兼容检查

- 目标：完成 SEO、性能、无障碍、安全、资源本地化和中国大陆访问兼容检查。
- 重点：关键公开页不得依赖 Google Fonts、Analytics、Maps、reCAPTCHA、YouTube 等境外资源。

## P9：部署、备份、恢复演练和最终验收

- 目标：选择境外低成本部署，配置域名/HTTPS/自动部署，完成备份、恢复演练和上线验收。
- 部署决策：第一版不办理 ICP；中国大陆访问目标为尽量兼容，不承诺绝对稳定。

## 已确认决策

- Astro + Keystatic；初期无传统数据库；内容使用 Markdown/MDX、YAML 或 JSON。
- GitHub 私人仓库由网站所有者本人控制；初期一个管理员；第一版接受 GitHub 登录，GitHub 账号必须启用双重验证。
- 默认中文；预留 `/zh/`、`/ja/`、`/en/`；第一版只要求中文完整上线。
- 联系方式只用邮箱链接和社交链接；不做表单、注册、评论、投稿、支付或社区。
- 少量图片可入 Git，但限制类型/尺寸/大小；大型视频、PDF、Office 和压缩包不得入 Git；媒体模型预留对象存储迁移。
- 年度基础预算目标不超过人民币 500 元；域名、仓库、内容和部署账号均由网站所有者控制。

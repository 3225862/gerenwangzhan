# P11 初始体检

## 通过项

- `npm ci`、lint、类型检查、测试、外部依赖检查和 build 均通过。
- Astro、Keystatic、Git 内容存储和静态构建架构保持不变。
- 草稿过滤、图片路径检查、关键外部依赖检查和密钥忽略规则存在。
- `/`、`/about/`、`/projects/`、`/articles/`、`/admin/` 和 `/keystatic/` 的入口已保留。

## 已修复项

- P0 测试项目已恢复为明确的草稿示例。
- P2 首页、个人资料和网站设置中的明显占位文字已替换为中性内容。
- 增加内容模板、Windows 本地运行指南、后台操作指南和部署路线。
- 补充公开链接检查报告和最终检查报告。

## 暂时接受的限制

- 没有真实域名、部署账号、GitHub OAuth 和生产 Secret，无法在本地替代真实上线。
- Keystatic 的高级字段仍是表单编辑，不是完整 Notion 富文本编辑器。
- 媒体库、批量上传、自动压缩、视频播放器和大文件对象存储未实现。
- 本地 `/keystatic` 需要开发模式；生产未启用 GitHub mode 时后台保持关闭。

## 不建议继续投入的高成本项

不建议在当前架构下继续仿造完整 WordPress 媒体库、数据库后台或自建复杂富文本服务。等真实内容和上线平台确定后，再根据实际维护痛点投入。
